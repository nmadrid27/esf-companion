# Templates

Blank forms for each ESF practice. Use the version that matches your situation.

---

## Position Statement

| File | Use when |
|------|----------|
| `position-statement.md` | Minimal setup. No frontmatter, no course fields. Works with any tool, including non-Claude-Code paths. |
| `position-statement-template.md` | Structured use with Claude Code. Includes frontmatter, course fields, and a post-session drift check. |

Both use the same three elements: My Stance, What Matters Most, What I Will Not Compromise On.

---

## Record of Resistance

| File | Use when |
|------|----------|
| `record-of-resistance.md` | Quick capture. Simple format, no frontmatter. |
| `record-of-resistance-template.md` | Structured capture with frontmatter. This is the format the ESF Companion uses when it creates Records of Resistance automatically. |

Both use the same section order: What AI Suggested → Why → What I Did Instead.

---

## AI Use Log

Three versions in order of complexity. Start with the one that fits your current project, and move up as your practice develops.

| File | Use when |
|------|----------|
| `ai-use-log.md` | Simple session tracking. One table per session with verification. Good for reconstruction on projects already in progress. |
| `ai-use-log-lite-template.md` | First projects. Simplified format with Five Questions check per session. Recommended for Level 1 (Discovery). |
| `ai-use-log-template.md` | Full version. Includes verification table, prompt evolution notes, intervention summary, and pattern analysis. Recommended from Level 3 (Independent Use) onward. |

---

## Other Templates

| File | Purpose |
|------|---------|
| `five-questions-checklist.md` | Standalone Five Questions audit. Run at major decision points, before submitting, or when something feels off. |
| `reflection-template.md` | End-of-project reflection. Includes a position summary, kept/revised/rejected section, Five Questions table, and disclosure. |
| `session-log-template.md` | Per-session record of decisions made, drift detected, Records of Resistance created, and next session plan. Auto-generated by the ESF Companion at session end. |
| `disclosure-statement.md` | Standalone disclosure. Short, standard, and detailed forms. |
| `evolution-log-template.md` | Documents how your ESF practice itself has changed across projects. Use after completing a project cycle to reflect on how your approach to AI collaboration has matured. Not a project artifact — a practice artifact. |
